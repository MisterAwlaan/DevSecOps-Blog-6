DevSecOps — Blog de Recettes
Patch Note · Avril 2026
Application Flask / SQLite · Dockerisée · Python 3.12

-----------------------------------------------------------------------------------------
Infrastructure & déploiement
-----------------------------------------------------------------------------------------

FIX
Adresse d'écoute Flask : 0.0.0.0 → 127.0.0.1
Par défaut, Flask se bind sur 0.0.0.0 ce qui expose l'app sur toutes les interfaces réseau. En environnement local, nous avons constaté que l'app était joignable depuis l'extérieur de la machine. 
Correction appliquée pour restreindre l'écoute à la boucle locale uniquement.
-----------------------------------------------------------------------------------------
INFRA
Dockerfile dupliqué : python:3.12-slim vs python:3.12-alpine
Deux fichiers de build coexistaient — un Dockerfile basé sur slim et un docker-compose.yml utilisant alpine. 
Le passage à Alpine a nécessité l'ajout de dépendances système (gcc, musl-dev, libffi-dev) pour compiler les packages Python natifs (Pillow notamment).
-----------------------------------------------------------------------------------------
INFRA
Lancement via python app.py au lieu de Gunicorn/WSGI
Le serveur de développement Flask est lancé directement via CMD ["python", "app.py"]. 
Cela déclenche le bloc if __name__ == '__main__': et utilise le serveur intégré, non recommandé en production mais retenu pour correspondre à l'environnement de TP.

-----------------------------------------------------------------------------------------
Sécurité — Upload de fichiers
-----------------------------------------------------------------------------------------

SEC
Validation MIME réelle des images uploadées via Pillow
La simple vérification d'extension (secure_filename) ne suffit pas — un fichier malveillant peut avoir l'extension .jpg mais contenir du code. 
Pillow est désormais utilisé pour ouvrir le fichier et vérifier son format réel. 
Si l'ouverture échoue ou que le format n'est pas dans la liste blanche (PNG, JPEG, GIF, WEBP), le fichier est supprimé du disque.
-----------------------------------------------------------------------------------------
SEC
Blocage explicite du SVG dans les uploads
Le SVG est un vecteur d'injection XSS connu (un fichier SVG peut embarquer du JavaScript). 
Il a été explicitement exclu de ALLOWED_EXTENSIONS et n'est pas non plus reconnu par Pillow dans ALLOWED_PIL_FORMATS, assurant un double blocage.
-----------------------------------------------------------------------------------------
SEC
Contrôle d'accès sur les recettes privées
Une route /recettes/<id> vérifie si la recette est marquée prive. 
Si oui, seul l'auteur connecté peut y accéder. Toute tentative non autorisée redirige vers l'accueil avec un flash danger.
-----------------------------------------------------------------------------------------
SEC
Hashage des mots de passe avec werkzeug
Les mots de passe ne sont jamais stockés en clair. 
L'inscription utilise generate_password_hash et la connexion check_password_hash. 
La modification du mot de passe en profil suit le même mécanisme.

-----------------------------------------------------------------------------------------
Fonctionnalités
-----------------------------------------------------------------------------------------

FEAT
CRUD complet des recettes (ajout, modification, suppression)
Chaque utilisateur connecté peut créer, modifier et supprimer ses propres recettes. 
Les routes vérifient que auteur_id == session['user_id'] avant toute modification, empêchant l'altération des recettes d'autrui.
-----------------------------------------------------------------------------------------
FEAT
Gestion du profil utilisateur avec photo de profil
L'utilisateur peut mettre à jour son mot de passe et changer sa photo de profil depuis la page /profil.
Les photos sont stockées dans static/img/profiles/ avec un préfixe unique par utilisateur (user_<id>).
-----------------------------------------------------------------------------------------
FEAT
Recettes publiques / privées
Lors de la création ou modification d'une recette, l'auteur choisit le statut public ou prive. 
La page d'accueil n'affiche que les recettes publiques via un filtre SQLAlchemy.
Connu / à surveiller
-----------------------------------------------------------------------------------------
NOTE
database.db commitée dans le repo
Le fichier SQLite est présent dans le dépôt Git. 
En production il faudrait l'ajouter au .gitignore et utiliser un volume Docker pour persister les données sans les exposer dans l'historique.
-----------------------------------------------------------------------------------------
NOTE
__pycache__ présent dans le repo
Le dossier de cache Python est versionné. Il devrait figurer dans .gitignore pour éviter de polluer l'historique avec des artefacts binaires.
-----------------------------------------------------------------------------------------
NOTE
Pas de werkzeug ni python-dotenv dans requirements.txt
app.py importe load_dotenv et utilise werkzeug.security, mais ces packages ne sont pas listés explicitement dans requirements.txt. 
Ils arrivent en dépendances transitives de Flask, mais les rendre explicites améliore la reproductibilité.
